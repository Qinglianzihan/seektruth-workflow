# 项目升级路线图

> 跨任务交接文件。本项目的"下一步该做什么"之来源。
>
> **新会话第一件事**：读这份文件 → 知道大方向 + 下一步该做什么。
>
> 与 `.stw/.progress.json`（单任务进度）配合，形成两层连续性：
> - `.progress.json` = 单任务做到第几阶段
> - `roadmap.md` = 整条路线做到第几件任务

> **⚠️ 派生文档警示**：本文件通常是对项目其它根文档（架构说明、方法论清单、设计决策）的**派生摘要**——
> 它自己也会漂：上游打了 ✓，本摘要可能忘了同步。
>
> STW 提供 `doc-drift` gate 做反向校验：扫本文件 `[x] **T{N}.**` 已完成项，
> 对照源文档查同编号是否仍挂"候选 / 待补 / **半** / 未实现"等过时字样，漂移即警告（非阻断）。
>
> 新会话首跑 `stw status`（漂移 > 0 时会提示一行），或直接 `stw check doc-drift` 看详情；
> `stw report` 归档后也会附带一次校验。

---

## 新会话标准协议

用户一句话："**接着推进**" 或 "**继续升级**"

AI 自动做：
1. 读本文件 → 知道总路线和已完成项
2. 跑 `stw status`（或 `rtk stw status`）→ 看是否有进行中任务
3. 有进行中 → 续上；无进行中 → 从下面"下一步"拎一条，跟用户确认后 `stw start`
4. 每完成一项 → 回来勾掉 `[x]` + 更新"下一步"

## 更新本文件的规则

- 完成一项 → `[ ]` 改 `[x]`，加归档行"最近完成"
- 发现新缺口 → 追加一条，标优先级
- 放弃某项 → 移到"暂不做"并写理由
- 本文件**不会**被 `stw init` 覆盖（已存在则自动归档旧版到 `.stw/history/`）
