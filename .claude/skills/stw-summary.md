# STW Phase 5: Summary

Thin trigger only. The current workflow rules live in `.stw/STW-Workspace.md`, `.stw/Summary-Template.md`, and CLI gates.

## Use

1. Fill the current summary template with outcome, changed files, tests, lessons, and follow-up risks.
2. Run `stw next` (or `rtk stw next` if available).
3. Run `stw report` if a report is needed.
4. Only then say the work is complete.

Keep summaries factual and reusable.
